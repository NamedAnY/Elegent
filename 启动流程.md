# 游戏启动流程

## 浏览器启动流程

<div class="section">


1. 使用 **VSCode F5** 启动，打开 `index.html`（可在 `.vscode/launch.json` 查看），加载 `index.js` 文件。  
2. `index.js` 加载所需库文件和编译文件 `bundle.js`。  
3. `bundle.js` 的入口函数是 `Main.ts`（最后执行 `new Main()` 激活启动类）。  
4. `Main.ts` 继承自 `fx.AppBase`，构造函数调用 `super(cfg, GameCfg.manifestFile)` → 执行 `onLoaded()` → `this.launch()` → 切换到指定启动场景。
</div>

## 微信平台启动流程

<div class="section">

1. `game.js` 为小游戏启动文件，加载：
  - `weapp-adapter.js`（平台适配）  
  - `first-screen.js`（首屏渲染）  
   同时加载 **loading 分包**（包含 logo、进度条），完成后调用 `pureLoading.js`。  
2. `pureLoading.js` 创建 **laya 节点** logo 与进度条，下载其他分包后调用 `index.js`：
  - loading 分包 → `game.js` 中加载  
  - 其他分包（`res`, `lib`, `sound`） → `pureLoading.js` 中加载  
  - 分包试玩 → 未验证（可能存在）  
3. `index.js` 加载库文件和 `bundle.js`。  
4. `bundle.js` 调用 `Main.ts`，最终切换场景。

</div>

# LoadingScript 初始化流程

## SDK 初始化

<div class="section">

1. 在 `onEnable()` 中调用 `loginSdk()`。  
2. `loginSdk()` 注册 `sdk.SdkEvent.E_SDK_INIT_OK` 并调用 `sdk.Sdk.instance.init()`。  
3. 不同平台的 `sdk.Sdk.instance` 对应不同 `sdk.xx.js` 文件，调用其 `init()`。  
4. `sdk.xx.js` 继承自 `fx.BaseLogic`，等同调用 `fx.BaseLogic.init()`。  
5. `fx.BaseLogin.init()` 调用 `initOnce()`，在 `sdk.xx.js` 中重写该方法 → 执行 SDK 初始化。

</div>

## 加载配置文件

<div class="section">

1. 调用 `initConfig()` → `Game.controller.queryVersionCfg()` 查询服务器配置。  
2. 查询到配置后更新 `Controller` 中的配置字段，供其它脚本访问。  

</div>

## 数据同步

<div class="section">

1. 同步配置后调用 `Game.user.parseFromLocal()` 读取本地数据：
  - 每个 info 依次读取本地 string 数据  
  - 将 string 转为 object  
  - 遍历 object，每个属性应用到对应 info  
2. 调用 `ServerStorage.I.init()` 获取服务器数据：
  - 根据用户信息登录服务器  
  - 登录后获取服务器存储的 string 数据  
  - 对比本地数据并同步，得到最终数据  
3. 最终数据写入：
  - `Game.user.saveToLocal()` 更新本地数据  
  - `Game.user.saveToServer()` 更新服务器数据

</div>
