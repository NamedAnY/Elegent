# 浏览器启动流程
<div class="section">

1. 使用 **VSCode F5** 启动，实际上打开的是 `index.html`（可在 `.vscode/launch.json` 中查看），在 `index.html` 中加载了 `index.js` 文件。  
2. 在 `index.js` 中加载所需库文件，以及加载编译的代码文件 `bundle.js`。  
3. `bundle.js` 的入口函数是 `Main.ts`（因为最后有 `new Main()` 激活启动类）。  
4. `Main.ts` 继承自 `fx.AppBase`，在其构造函数中会调用 `super(cfg, GameCfg.manifestFile)`。父类 `fx` 构造函数处理完成后会调用 `Main.ts` 中的 `onLoaded` 方法，`onLoaded` 方法中会调用 `this.launch()`，最终 `fx` 会切换到指定启动场景。  

</div>

# 微信平台启动流程
<div class="section">

1. `game.js` 为小游戏启动文件，在其中会调用  
   - `weapp-adapter.js`（平台适配文件）  
   - `first-screen.js`（首屏渲染文件）  
   然后加载 **loading 分包**（包含 logo、进度条等），加载完成后调用 `pureLoading.js`。  

2. 在 `pureLoading.js` 中会用 **laya 节点**创建 logo、进度条，并下载其他分包，分包下载完成后调用 `index.js`。  
   - `game.json` 中的分包可分为三类：  
     - loading 分包 → 在 `game.js` 中加载  
     - 其他分包（`res`, `lib`, `sound`）→ 在 `pureLoading.js` 中加载  
     - 分包试玩 → 未验证（可能存在）  

3. 在 `index.js` 中加载所需库文件，以及我们编译的代码文件 `bundle.js`。  
4. `bundle.js` 的入口函数是 `Main.ts`（因为最后有 `new Main()` 激活启动类）。  
5. `Main.ts` 继承自 `fx.AppBase`，在其构造函数中会调用 `super(cfg, GameCfg.manifestFile)`。父类 `fx` 构造函数处理完成后会调用 `Main.ts` 中的 `onLoaded` 方法，`onLoaded` 方法中会调用 `this.launch()`，最终 `fx` 会切换到指定启动场景。  

</div>
